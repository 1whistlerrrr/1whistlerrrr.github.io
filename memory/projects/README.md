# Projects —— 项目登记表（按项目组织）

> 记忆以**项目为中心**：每个项目/研究一个文件夹，自包含它的全生命周期——
> 工作交接（`handoff.md`）、路线图（`roadmap.md`）、调研（`research.md`）等。
> 新会话先看本表定位项目，再进对应文件夹读 `handoff.md`。

## 每个项目文件夹放什么

| 文件 | 记忆类型 | 内容 |
|------|---------|------|
| `handoff.md` | 短期/工作记忆 | 当前做到哪、下一步、踩过的坑（**每次先读**） |
| `roadmap.md` | 项目记忆 | 阶段规划、实施路线（可选） |
| `research.md` | 项目记忆 | 前期调研、资料汇总（可选） |
| 其他 | 按需 | 该项目的一系列流程文件 |

只有 `handoff.md` 是必需的；roadmap / research 按项目需要添加。

## 约定

- 新项目：`memory/projects/<项目-slug>/` 建文件夹，至少放一份 `handoff.md`，然后在上面的总览表加一行
- 项目 slug 小写连字符，别太细碎——按**项目**粒度，不是按单次任务
- 项目完成后把状态改成 ✅，文件夹保留供追溯，不要删

## handoff.md 模板

```markdown
# HANDOFF —— <项目名>

> **归属**：<项目 | 研究> · <名称>
> **状态**：<🟢/🟡/✅/🔴> · 最后更新 YYYY-MM-DD

## 一、目标 / 要做的决策
## 二、已完成
## 三、当前状态（待解决 / 待验证，复选框）
## 四、下一步（按优先级）
## 五、踩过的坑
## 六、关键路径（绝对路径）
```
