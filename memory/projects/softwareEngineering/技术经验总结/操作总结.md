## Git操作总结
### 1.Merge新的更改
#### 1.1新建临时分支进行merge
1. `git checkout -b temp/feat-partial 目标提交哈希or目标分支名~6`  #（-b创建一个新分支，并且自动切换到这个新分支上）
2. `git checkout develop`   # 切换当前分支
3. `git merge --no-ff temp/feat-partial -m "merge: 合入feat分支的登录校验模块"` 
4. `git branch -d temp/feat-partial`  # 安全删除：已经合并了的分支会被删除 -D强制删除
#### 1.2将当前分支进行merge ⭐️
1. `git checkout develop` 
2. `git pull origin develop` # 先切换分支然后再拉取 
3. `git merge --no-ff feat/MS -m "merge: 合人物料安全管理完整功能"` # --no-ff 表示如果可以直接和也不要这么做
4. `git push origin develop` # 推送上去

### 2.将当前分支的某几个更改提取到另一个分支上
1. 获取当前提交哈希`git log --oneline`
2. 切换到目标分支 `git checkout 目标分支名`
3. 迁移两个提交 `git cherry-pick 较早的commit哈希 较晚的commit哈希` 最近的 2 个连续提交`git cherry-pick HEAD~2..HEAD`

### 3.冲突！
1. 查看当前状态`git status`
``` 
<<<<<<< HEAD    // 分割线以上：你当前所在分支（目标分支）的代码
本地/当前分支的代码内容
=======         // 中间分割线
合入/远程分支的代码内容
>>>>>>> feat/MS // 分割线以下：你要合进来的源分支名
```
2. 将修改好的文件添加进去`git add .`
3. `git commit` Git 会自动生成本次合并的提交信息（包含合入分支、冲突解决的记录）

### 4.rebase的操作
1. rebase的操作git fetch zny master && git rebase --autostash zny/master  == git pull --rebase zny master

## 本地调试总结
### 命令行
#### 1.杀死当前某个进程
1. 查找端口： `lsof -i:5050`
2. 杀死对应的PID ： `kill 1234`
快速删除某个端口：`lsof -ti :5050 | xargs kill -9`
